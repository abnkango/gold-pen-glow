# خطة لوحة تحكم Admin — ورقة وقلم

مشروع Lovable **منفصل** بنفس الهوية البصرية (ذهبي/أسود) لإدارة كل محتوى التطبيق. أنت الآدمن الوحيد. التخزين عبر Cloudflare (Stream للفيديو + R2 للملفات).

---

## 1. هيكل البيانات (Lovable Cloud / Postgres)

```text
branches (الفروع)         → scientific | literary | ninth
  └─ subjects (المواد)    → رياضيات، فيزياء... (مرتبطة بفرع)
       └─ units (الوحدات) → الفصل الأول، الثاني...
            └─ lessons (الدروس) → اسم الدرس + ترتيبه
                 ├─ videos      (Cloudflare Stream UID)
                 ├─ pdfs        (R2 file key + اسم ظاهر)
                 └─ quizzes     (أسئلة + إجابات)

teachers (الأساتذة)       → الاسم، الصورة، الفرع، المواد التي يدرّسها، حالة الظهور
books (الكتب)             → PDF عام (مرتبط بفرع/مادة/درس)، اسم ظاهر
students (auth.users)     → قراءة فقط: عدد المسجلين، آخر دخول، الفرع المختار
admin_users               → جدول منفصل لحساب الآدمن (دور admin فقط)
```

كل عنصر له حقول: `is_visible` (يظهر/مخفي في التطبيق)، `display_order` (ترتيب الظهور)، `display_name` (الاسم الذي يراه الطالب).

## 2. شاشات لوحة التحكم

| الشاشة | الوظيفة |
|---|---|
| **تسجيل دخول** | إيميل + كلمة سر، محمي بـ role=admin |
| **الرئيسية (Dashboard)** | إحصائيات: عدد الطلاب المسجلين، عدد الدروس/الفيديوهات/الكتب لكل فرع، آخر الأنشطة |
| **الفروع** | 3 فروع ثابتة (علمي/أدبي/تاسع) — كل واحد بوابة مستقلة |
| **داخل كل فرع → المواد** | إضافة/تعديل/حذف مادة، أيقونة، اسم، ترتيب، إظهار/إخفاء |
| **داخل كل مادة → الوحدات** | إضافة وحدة، اسم، ترتيب |
| **داخل كل وحدة → الدروس** | إضافة درس، اسم ظاهر، ترتيب، إظهار/إخفاء |
| **داخل كل درس** | تبويبات: **الفيديو** \| **اختبار** \| **كتب PDF** |
| ↳ **رفع فيديو** | Upload مباشر لـ Cloudflare Stream → حفظ الـ UID + اسم |
| ↳ **رفع PDF** | Upload لـ R2 → حفظ الـ key + اسم ظاهر |
| ↳ **بناء اختبار** | محرر أسئلة (multiple choice) + إجابة صحيحة + شرح |
| **الأساتذة** | CRUD: صورة، اسم، فرع، مواد، وصف، حالة "متاح للتواصل" |
| **الكتب (مكتبة عامة)** | رفع PDF + ربطه بفرع/مادة، اسم ظاهر |
| **الطلاب** | قراءة فقط: قائمة + بحث + فلترة حسب الفرع + إحصائيات |
| **الإعدادات** | تغيير كلمة سر، إعدادات الفروع |

## 3. التكامل مع Cloudflare

- **Cloudflare Stream**: رفع الفيديو من اللوحة → يرجع UID → نخزّنه في DB. التطبيق يشغّل الفيديو بـ HLS تلقائياً، مع حماية Signed URLs (لا أحد يقدر يحمّل الفيديو).
- **Cloudflare R2**: تخزين ملفات PDF والصور بسعر رخيص (لا توجد رسوم خروج بيانات).

**ما أحتاجه منك لاحقاً (ليس الآن — أبني الواجهة أولاً):**
1. `CLOUDFLARE_ACCOUNT_ID`
2. `CLOUDFLARE_STREAM_TOKEN` (من Cloudflare → My Profile → API Tokens → Stream)
3. `CLOUDFLARE_R2_ACCESS_KEY_ID` + `CLOUDFLARE_R2_SECRET_ACCESS_KEY` + اسم الـ bucket
4. `CLOUDFLARE_STREAM_SIGNING_KEY` (لتوقيع روابط الفيديو المحمية)

سأشرح لك خطوة بخطوة كيف تحصل عليها عند الوصول لمرحلة الربط.

## 4. الأمان

- جدول `admin_users` منفصل عن `students`، RLS صارم: فقط صاحب دور admin يقدر يكتب.
- كل العمليات الحساسة (رفع، حذف) عبر server functions مع `requireSupabaseAuth` + فحص الدور.
- رفع الفيديو يمر عبر Cloudflare Direct Upload URL (لا يمر عبر سيرفرنا → لا حدود حجم).

## 5. مراحل التنفيذ

**المرحلة 1 (الآن):** بناء الواجهة الكاملة + Lovable Cloud + جداول DB + auth الآدمن + شاشات CRUD للمواد/الوحدات/الدروس/الأساتذة/الكتب — بدون رفع فيديو حقيقي (نضع placeholder لـ Stream UID يدوياً).
